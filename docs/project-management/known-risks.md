# Known Risks

Parent: `docs/project/ventureos-system-bible.md`

Source Of Truth: This file is the source of truth for project-management risk tracking.

## Current Risks

- Implementation may begin before phase approval.
- Documentation may drift if source-of-truth boundaries are not maintained.
- Gateway requirements need future implementation detail before external access.
- Runtime documentation is initial and requires deeper Phase -1 architecture specification.
- Decision Ledger and Evidence Ledger specs are restored but not yet mapped to future database or API contracts.
- Phase 1 may be mistaken for implementation approval unless the approval boundary remains explicit.
- Technology direction may be mistaken for integration authorization.
- Single-venture assumptions may reappear unless Portfolio-first architecture is enforced.
- Domain ownership may remain vague unless initial capabilities are mapped to Domains.
- Event volume and retention requirements may be underestimated.
- Resource consumption and cost tracking may be postponed too long.
- Founder Operating Console terminology may drift back to generic dashboard language.
