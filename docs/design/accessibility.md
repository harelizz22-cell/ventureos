# Accessibility

Parent: `docs/design/README.md`

Source Of Truth: This file is the source of truth for VentureOS accessibility direction.

## Purpose

Define accessibility expectations for the design foundation before UI implementation begins.

## Rules

- Text must be readable on dark surfaces.
- Semantic colors must not be the only way to communicate state.
- Important status, approval, and governance information must be available as text.
- Focus states must be visible.
- Interaction targets must be usable in desktop and companion mobile contexts.
- Tables and timelines must remain navigable and understandable.

## Open Decisions

- Exact contrast targets.
- Keyboard navigation model.
- Screen reader labeling conventions.
