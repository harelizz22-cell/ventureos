# Design Principles

Parent: `docs/design/README.md`

Source Of Truth: This file is the source of truth for VentureOS design principles.

## Core Principle

Function defines form.

## Principles

- Design must clarify decisions, not decorate activity.
- Founder attention is scarce and must be protected.
- Evidence, governance, risk, and decisions must be visually legible.
- The interface should feel calm, premium, precise, and executive.
- Desktop workflows are primary; mobile supports monitoring and companion review.
- The product must not resemble a generic AI dashboard.
- Visual hierarchy must communicate operational priority.
- Governance states must be clear without visual alarmism.
- Product language should favor Venture, Operation, Decision, Workflow, Evidence, and Governance.

## Avoid

- Playful design language.
- Neon palettes.
- Cyberpunk styling.
- Glassmorphism.
- Excessive ALL CAPS.
- "Mission" as the primary product term.
- Decorative color usage.
