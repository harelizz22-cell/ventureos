# Interaction Model

Parent: `docs/design/README.md`

Source Of Truth: This file is the source of truth for VentureOS interaction model direction.

## Purpose

Define how founder and operator interactions should feel and behave at a high level.

## Interaction Direction

- Interactions should be deliberate.
- Approvals should require clarity before commitment.
- Governance blocks should explain cause and next valid path.
- Evidence review should stay close to decisions.
- Operational workflows should minimize unnecessary navigation.
- Destructive or irreversible actions must be strongly confirmed.

## Command Behavior

Commands should reveal policy status, required approvals, and expected consequences before execution.

## Feedback Behavior

Feedback should be quiet, precise, and state-based. Avoid celebratory or playful feedback patterns.
