# Layout System

Parent: `docs/design/README.md`

Source Of Truth: This file is the source of truth for VentureOS layout direction.

## Product Posture

VentureOS is browser-first and desktop-primary. Mobile is a companion experience for monitoring, alerts, and lightweight review.

## Core Screen Structure

Layer 1: Founder Status

Layer 2: Decision Layer

Layer 3: Operational Layer

## Layout Rules

- Founder status should be visible before operational detail.
- Decisions should be visually distinct from activity streams.
- Operational views should support scanning, comparison, and repeated use.
- Dense information is acceptable when organized and governed by hierarchy.
- Avoid marketing-style hero layouts inside the operating product.
- Avoid card-heavy decoration that hides operational priority.

## Mobile Rule

Mobile must not define the primary product shape. It should support companion actions without replacing desktop governance workflows.
