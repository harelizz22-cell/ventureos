# Compliance Rules

Parent: `docs/project/ventureos-system-bible.md`

Source Of Truth: This file is the source of truth for compliance review rules.

## Purpose

Compliance rules define how work is checked against architecture, governance, security, recovery, and documentation requirements.

## Rules

- Meaningful work must be auditable.
- Governance exceptions require explicit approval.
- Architecture-changing work requires ADR compliance.
- Production asset work requires recovery readiness.
