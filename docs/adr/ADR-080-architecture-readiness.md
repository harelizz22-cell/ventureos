# ADR-080: Architecture Readiness

Status: Accepted

Date: 2026-07-01

## Context

Before implementation planning, VentureOS needs measurable readiness criteria across architecture categories.

## Decision

Adopt Architecture Readiness categories: Governance, Security, Runtime, Knowledge, Treasury, Policy, Evidence, AI, Portfolio, Operations, and Documentation.

Each category must define readiness criteria, success criteria, and blocking conditions.

## Rationale

Readiness review needs explicit criteria rather than subjective confidence.

## Consequences

- Architecture readiness informs internal and external review.
- Readiness does not authorize implementation.
- Blocking conditions must be resolved or explicitly accepted before implementation approval.

## Enforcement

Future implementation approval must reference Architecture Readiness, Architecture Scorecard, Phase 2 Completion Checklist, and accepted review outcomes.
