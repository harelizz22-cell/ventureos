# ADR Workflow

Parent: `docs/project/ventureos-system-bible.md`

Source Of Truth: This file is the source of truth for ADR workflow.

## Purpose

Architecture Decision Records document approved or proposed architecture decisions.

## Workflow

1. Identify decision.
2. Gather evidence.
3. Update architecture where needed.
4. Create or update ADR.
5. Update phase document if implementation is affected.
6. Seek required approval.

## Rule

Architecture-changing work requires an ADR before implementation.
