# ADR-073: Milestone Funding

Status: Accepted

Date: 2026-07-01

## Context

Large up-front releases increase capital risk and reduce the Founder ability to stop, pause, or reallocate when evidence changes.

## Decision

Adopt Milestone Funding so capital is released in stages through Budget Approved -> Milestone -> Review -> next Milestone -> Completion.

Milestone Funding supports partial funding, holdbacks, re-budgeting, fund cancellation, and unused capital return.

## Rationale

Staged release preserves capital discipline and ties funding to evidence and progress.

## Consequences

- Capital release must link to milestone evidence.
- Partial release does not imply future release approval.
- Cancelled or unused capital must return through governed lifecycle transition.

## Enforcement

Every milestone release must pass Policy Engine, Treasury lifecycle verification, and required approvals before release.
