# ADR-015: Enterprise Value Engine

Parent: `docs/adr/README.md`

Source Of Truth: This file is the source of truth for the accepted Enterprise Value Engine decision.

## Status

Accepted

## Context

Architecture Audit #005 expands VentureOS from a Venture Builder into an Enterprise Value Operating System.

## Decision

VentureOS exists to maximize long-term Enterprise Value while preserving Founder control, governance, transparency, and capital discipline.

Revenue is an important KPI. Enterprise Value is the primary objective.

The Enterprise Value Engine is adopted as a first-class architecture concept.

## Consequences

- Enterprise Value becomes the highest-level optimization objective.
- Revenue remains an important KPI but is not the primary objective.
- Business Intelligence, Value Graph, Capital Allocation, Investment, and Venture Health must support Enterprise Value evaluation.

## Enforcement

Future architecture and phase documents must treat Enterprise Value as the primary optimization objective.
