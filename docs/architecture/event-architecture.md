# Event Architecture

Parent: `docs/architecture/ventureos-architecture.md`

Source Of Truth: This file is the source of truth for event architecture.

## Purpose

Events represent meaningful system facts that other components may observe.

## Event Categories

- Evidence recorded.
- Decision proposed.
- Approval requested.
- Approval granted.
- Policy denied.
- Workflow started.
- State changed.
- Asset registered.
- Incident opened.

## Rule

Event contracts must be documented in `docs/contracts/event-contract-template.md` before implementation.
