# Milestone Capital Release

Parent: `docs/architecture/ventureos-architecture.md`

Source Of Truth: This file is the source of truth for Milestone Capital Release architecture.

## Purpose

Milestone Capital Release prevents large funding amounts from being released all at once by tying future capital release recommendations to milestone evidence, governance, and compliance status.

This is future architecture readiness only. It does not authorize VentureOS to hold funds, transfer funds, or execute regulated financial activity.

## Rules

- Funding is divided into milestones.
- Every milestone has success criteria.
- Every capital release requires evidence.
- Failed milestones may pause or stop the Venture.
- Unused capital handling must be defined by legal/compliance structure.
- Capital release is always governed.
- Milestone Funding is the adopted Phase 2 source for staged release, partial funding, holdbacks, re-budgeting, cancellation, and unused capital return.
- Treasury Domain owns capital release state and lifecycle tracking.

## Release Review

Capital release recommendations must connect to milestone success criteria, Evidence, Decision, Audit, budget plan, governance status, legal/compliance status, and expected Enterprise Value impact.

## Rule

Milestone Capital Release may recommend release, pause, stop, or review actions. It must not move money or execute regulated financial activity.

No money can move without Policy Engine evaluation and Treasury lifecycle validation.

## Placeholder Status

This is a future-phase architecture placeholder. It does not define fund custody, payment processing, securities workflows, escrow mechanics, schemas, integrations, or implementation.
