# Decision Ledger Specification

## Purpose

The Decision Ledger records meaningful decisions and their supporting rationale.

## Required Fields

- Decision ID
- Title
- Owner
- Status
- Options considered
- Evidence links
- Decision
- Approval record
- Expected outcome
- Revisit trigger
- Rollback criteria
- Created at
- Updated at

## Rules

- Meaningful decisions require evidence links.
- Approval status must be explicit.
- Revisit or rollback criteria must be defined where relevant.
