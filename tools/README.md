# Tools

Parent: Repository root.

## Purpose

This folder is reserved for internal project tools that support approved development, review, validation, or operational workflows.

## What Belongs Here

- Approved internal tooling.
- Developer utilities.
- Review and validation helpers.
- Tooling prototypes explicitly authorized by an approved phase.

## What Does Not Belong Here

- Production application code.
- Autonomous execution components.
- Secrets, credentials, or environment-specific values.
- Architecture decisions or source-of-truth documentation.

## Current Status

Reserved for future implementation.
